# 项目记忆

累计反思 10 次

## 遗留问题与待办

1. **重复检测需加强证据链**：必须提供对比摘要或置信度，否则宁可不输出重复判断。
2. **分类与动作对齐**：推荐动作应与最终分类一致；需求已实现时优先 `question`/`documentation`。
3. **标签完整性校验**：基于推荐动作自动映射标签（如"仅改文档"→ `documentation`）；已满足的需求不标高置信度 `enhancement`。
4. **标题优化机会**：原标题含歧义或冗余时主动建议改写；≤5 汉字且无具体对象时必须改写。
5. **回归缺陷识别**：对 Mixin 继承链遗漏问题，建议添加自动化导入测试 + CI 拦截。
6. **优先级结合业务影响**：绕过审核/漏报场景应倾向 `high`，而非仅看技术复杂度。
7. **系统依赖降级策略**：引入外部库需提供 `ImportError` 捕获与降级方案。
8. **可行性方案分层**：提供"最小可行实现"与"高级实现"两阶段选项。

## 关键经验

- **复用已有基础设施**（ISSUE27）：直接复用 anti_flood 消息队列做片段拼接检测，避免重复造轮子。
- **风险与缓解措施成对出现**：每个风险附带解决思路，便于快速评估可落地性。
- **需求已满足的分类处理**（ISSUE24/31）：功能已实现时归为 `question`/`documentation`，而非 `enhancement`。
- **短消息拼接检测设计**（ISSUE27）：采用"连续短消息+间隔<5s"；拼接前过滤标点；非文本消息打断窗口。
- **系统依赖敏感性**（ISSUE26）：外部库安装复杂度影响 good first issue 判定，需标注环境要求。
- **二维码解码优先于 OCR**（ISSUE26）：先轻量解码再 OCR；解码 URL 应过 AC 自动机域名黑名单。
- **Mixin 继承同步维护**（ISSUE18/19）：`Main` 类继承列表遗漏 `CommandsMixin` 导致 6 个核心命令崩溃；MRO 为"左高右低"。
- **OneBot 协议约束**：撤回窗口 2 分钟、禁言最小粒度 60 秒，可行性分析必须引用。
- **踢人入口全量扫描**（ISSUE15）：4 个踢人路径（指令/LLM/远程/审核），需统一重构共享方法。
- **AC 自动机 + 正则回退**：67k 词库性能优化方案，关键词匹配 O(n)。
- **配置设计范本**：`kick_recall_enabled` + `kick_recall_count` 命名规范、类型明确。
- **用户体验优先**：即使技术可行，若与用户直觉冲突（如秒改分），应坚持用户直觉。

## 仓库特有模式

- 多 Mixin 集中继承（`ModerationMixin`, `AntiFloodMixin` 等），需维护 Mixin 清单。
- 审核流程：AC 自动机初筛 → LLM 二次判断 → 撤回 + 说明。
- SQLite 存储（审核日志 + 外置词库 `lexicon.db`），WebUI 迁移助手。
