# 项目记忆

累计反思 25 次

## 遗留问题与待办

1. **重复检测证据链+范围声明**：输出重复判断必须提供对比摘要或置信度；无重复时也需附检索范围/阈值说明。
2. **分类-标签-动作一致性**：分类应与推荐动作和标签对齐（垃圾 Issue 归 `other`/`invalid`，功能已存在归 `question`+`documentation`）。
3. **标签完整性校验**：基于推荐动作自动映射标签；已满足的需求不标高置信度 `enhancement`。
4. **标题优化常态化**：原标题含歧义/冗余/≤5字时主动改写，加分类前缀。
5. **优先级结合业务影响**：绕过审核/漏报/提权场景倾向 `high`，信息索取类统一 `low`。
6. **系统依赖降级策略**：引入外部库需提供 `ImportError` 捕获与降级方案。
7. **可行性方案分层**：提供"最小可行实现"与"高级实现"两阶段选项。
8. **截图 Issue 响应流程**：信息不足时优先 `question`，枚举可能区域，要求用户文字确认。
9. **协议拼接字段通用剥离**：推动通用 `strip_protocol_fragments` 函数。
10. **critical 标签阈值从严**：仅安全绕过、全群异常、数据泄露等标 0.8+。
11. **垃圾/Spam Issue 流程**：分类 `other`/`invalid`，标签 `invalid`，不进行可行性分析，直接关闭；频繁出现时建议自动化过滤。

## 关键经验

- **功能已存在场景标准模板**（ISSUE45）：`question`+`documentation` 双标签，低优先级，附三层建议（告知用户→确认配置→扩展方向）。
- **垃圾 Issue 标准处置**（ISSUE48）：纯推广 Issue 直接关闭+标记 `invalid`；标题无信息量时强制改写；重复检测需声明范围。
- **文档醒目性关注**（ISSUE45）：同类咨询反复出现时主动建议改进 README 布局，从根源减少疑问。
- **需求已满足的分类处理**（ISSUE24/31/40）：功能已实现时归 `question`/`documentation`，`enhancement` 置信度降至 0.5~0.6。
- **Mixin 继承同步维护**（ISSUE18/19）：`Main` 类遗漏 `CommandsMixin` 导致崩溃；MRO 为"左高右低"。
- **OneBot 协议约束**：撤回窗口 2 分钟、禁言最小粒度 60 秒，可行性分析必须引用。
- **踢人入口全量扫描**（ISSUE15）：4 个踢人路径需统一重构共享方法。
- **协议拼接字段剥离**（ISSUE33/41）：审核时剥离 OneBot 拼接内容，仅匹配用户纯输入。
- **good first issue 门槛把控**（ISSUE35）：避免推荐给多文件修改、配置兼容或权限理解门槛高的问题。
- **语气滤镜 + 复用历史证据**（ISSUE37）：情绪化标题先提取纯需求；引用 CHANGELOG 历史变更提升可信度。
- **低质标签过滤规则**（ISSUE39）：噪声标签（置信度<0.2）应配过滤规则剔除。
- **短消息拼接检测设计**（ISSUE27）：连续短消息+间隔<5s；拼接前过滤标点；非文本消息打断窗口。

## 仓库特有模式

- 多 Mixin 集中继承（`ModerationMixin`, `AntiFloodMixin` 等），需维护 Mixin 清单。
- 审核流程：AC 自动机初筛 → LLM 二次判断 → 撤回 + 说明。
- SQLite 存储（审核日志 + 外置词库 `lexicon.db`），WebUI 迁移助手。
