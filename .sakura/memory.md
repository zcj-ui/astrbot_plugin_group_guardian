# 项目记忆

累计反思 5 次

## 遗留问题与待办

1. **重复检测需加强证据链**：多次出现"可能是 #XX 的重复"但无依据，应提供对比摘要或置信度 <0.7 不输出。
2. **分类与动作对齐**：分析结论的推荐动作应与最终分类一致（如"仅改文档"→ `documentation`），避免混淆。
3. **标签完整性校验**：基于推荐动作补充缺失标签（如 `duplicate`、`regression`、`documentation`）。
4. **标题优化机会**：原标题含歧义或冗余时，应主动提供更精确的改写建议。
5. **回归缺陷识别**：对 Mixin 继承链遗漏问题，建议添加自动化导入测试 + CI 拦截。

## 关键经验

- **Mixin 继承同步维护**（ISSUE18/19）：`Main` 类继承列表遗漏 `CommandsMixin` 导致 6 个核心命令崩溃；MRO 为"左高右低"，修复时需注意顺序。
- **OneBot 协议约束**：撤回窗口 2 分钟、禁言最小粒度 60 秒，可行性分析必须引用。
- **踢人入口全量扫描**（ISSUE15）：4 个踢人路径（指令/LLM/远程/审核），功能实现需统一重构共享方法。
- **AC 自动机 + 正则回退**：67k 词库性能优化方案，关键词匹配 O(n)。
- **配置设计范本**：`kick_recall_enabled` + `kick_recall_count` 命名规范、类型明确。
- **用户体验优先**：即使技术可行，若与用户直觉冲突（如秒改分），应坚持用户直觉。

## 仓库特有模式

- 多 Mixin 集中继承（`ModerationMixin`, `AntiFloodMixin` 等），需维护 Mixin 清单。
- 审核流程：AC 自动机初筛 → LLM 二次判断 → 撤回 + 说明。
- SQLite 存储（审核日志 + 外置词库 `lexicon.db`），WebUI 迁移助手。
