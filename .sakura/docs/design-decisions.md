# 设计决策记录

## 一、Mixin 组合模式

**决策**：使用多 Mixin 集中继承于 `Main` 类，而非单一庞大类。
**原因**：模块职责单一，降低耦合，便于功能独立开发和测试。
**风险**：继承列表遗漏某 Mixin 将导致部分命令运行时崩溃（如 ISSUE18/19）。
**缓解**：添加自动化导入测试 + CI 拦截，确保 `Main` 实例可调用所有命令。

## 二、AC 自动机 + 正则回退

**决策**：核心词库匹配采用 AC 自动机（Aho-Corasick），支持 67k 词库 O(n) 匹配。
**补充**：正则表达式作为回退机制，处理 AC 自动机无法表达的复杂模式。
**存储**：外置词库 `lexicon.db`（SQLite），WebUI 提供迁移助手。

## 三、短消息拼接检测

**方案要点**：
- 复用 `anti_flood` 消息队列（按群×用户维度存储）
- 仅对**连续短消息**生效，间隔 < 5s
- 拼接前过滤标点/空白符
- 非文本消息（图片、表情）打断拼接窗口
- 默认仅词库初筛（不启用 LLM 二次判断），避免误伤
- 惩罚独立于防刷屏，检测命中后清空该用户片段计数

## 四、二维码解码策略

**优先级**：先轻量解码（pyzbar）→ 再 OCR（作为兜底）
**安全**：解码提取的 URL 需过 AC 自动机域名黑名单
**降级**：无 zbar 环境时捕获 `ImportError`，自动降级为纯 OCR 流程

## 五、踢人入口统一

**4 个踢人路径**：指令踢人、LLM 审核踢人、远程跨群踢人、审核流程踢人
**重构决策**：提取共享方法，避免每个入口独立实现导致不一致
